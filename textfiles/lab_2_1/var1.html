Обґрунтування вибору адаптивного макету

1. Чому адаптивний макет?

Адаптивний макет (responsive layout) дозволяє вебсайту коректно відображатися на різних пристроях: від мобільних телефонів до великих екранів моніторів. Це досягається завдяки використанню гнучких сіток, медіазапитів у CSS та адаптивних зображень.

2. Приклади реалізації:

  • Viewport: Мета-тег <meta name="viewport" content="width=device-width, initial-scale=1.0"> дозволяє браузеру автоматично масштабувати сторінку.
  • Гнучкі розміри: Використання відсотків або одиниць vw, vh для ширини та висоти.
  • Медіазапити: CSS-медіазапити, які дозволяють застосовувати різні стилі залежно від ширини екрану.

3. Переваги адаптивного макету:

  • Забезпечує зручність користувачам незалежно від їхнього пристрою.
  • Сприяє SEO-оптимізації, оскільки пошукові системи віддають перевагу мобільним сайтам.
  • Економія часу та ресурсів на розробку окремих версій для мобільних і десктопних пристроїв.

Базова концепція використаного коду

При верстці сторінок засобами CSS у наведеному коді використовується наступна базова концепція:

1. Семантична структура HTML

Код використовує семантичні теги, такі як <header>, <section>, <footer>, щоб забезпечити логічну та зрозумілу структуру сторінки. Це сприяє зручності підтримки коду, покращує SEO та забезпечує доступність для людей з обмеженими можливостями.

2. Принцип Mobile-First

Код створено з урахуванням мобільних пристроїв: стилі базуються на простішому вигляді для малих екранів, а для більших екранів використовуються медіазапити. Це забезпечує адаптивний дизайн і зручність користування на різних пристроях.

3. Використання модульності

Кожен блок сторінки, такий як "Головна секція", "Формат навчання", "Викладачі", поділений на окремі <section>. Це дозволяє легко додавати або змінювати окремі елементи без впливу на інші частини сторінки.

4. Гнучкий дизайн

Код містить інтерактивні елементи, наприклад, форми, таблиці, кнопки, що забезпечують зручну взаємодію з користувачем. Також використовується CSS для естетичного вигляду та чіткого відображення даних.

5. Використання зображень

Для підвищення візуальної привабливості використовуються зображення (<img>) та іконки. Це робить інтерфейс більш зрозумілим і привабливим для користувача.

6. Покращення доступності

У формі додані необхідні атрибути, наприклад, required, щоб уникнути помилок при введенні даних.
